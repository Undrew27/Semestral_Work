Intro

The idea of this project is a demonstration of working with markup languages. As an example, the sample of a simple database of university students was chosen (specifically the CZU example). The records contain some basic information about the users who are registered in the university system. Since it is only a demonstration, the database contains only a few records and can not be expected to have a complete list of students at that time.

The task contains these files:
czu.xml
An XML document that contains the student's dataset structure.
czu.xsd
XSD document with XML Schema Definition, which defines resp. Describes the structure of the previous XML document.
czu.xslt
XSLT document (eXtensible Stylesheet Language Transformations), a prescription in the recommended styling language for XML documents.
czu.html
The resulting HTML document after applying the above mentioned XSLT transform.
dokumentace.html
Documentation for the project

Methodology

Some online tools have been used to make it easier to work with markup languages. Using these tools, for example, the validated XML and HTML files were validated. The XSD generator was also used. The resulting XSD was then manually adjusted to suit your needs. For XSLT transformations, a tool was also used to convert source XML into the desired HTML format. Below is a list of used tools along with web links.

  • XML Validator
  • XSD/XML Schema Generator
  • XML Validator - XML Schema (XSD)
  • XSL Transformer - XSLT
  • HTML Validator

Description of elements and attributes

Used elements are divided into 4 logical sections"

1. Personal Information

The section where the student's basic personal information is stored

Title
Element in which the recorded title is included (data type STRING)
Attributes: position - Defines the position of the title - BEFORE or AFTER by name
Name
Name of the student (data type STRING)
MiddleName
MiddleName of the student (data type STRING)
Surname
Surname of the student (data type STRING)
BirthDate
Keeps date of birth (data type DATE)
Attributes: format - Indicates the format of the date value stored
PersonalID
PersonalID of the student (own data type Personal_ID)
Sex
Sex of the student (own data type SEX)
Nationality
Student nationality (data type STRING)
Height
Height of the student(data type INTEGER)
Weight
Weight of the student(data type INTEGER)
MaritalStatus
Marital status of the student(data type BOOL)
NumberofChildren
Number of children of the student(data type INTEGER)

2. Contact Information

The sections where the contact information of the student is stored

Street
Student address - Street name without number (data type STRING)
Number
Attributes: address - Indicates the number of the house / apartment and three types are distinguished: descriptive, registration or orientative
City
Student address - Name of municipality (data type STRING)
PartofCity
Student address - Community part specifications (data type STRING)
PSC
Postcode (own PSC data type)
State
Student address - State name (data type STRING)
Telephone
Student telephone contact (data type STRING)
Attributes: type - Each student may have more types of phone numbers
Email
Student e-mail contact (data type STRING)
Attributes: priority - Each student can have more contact emails, and the contact priority is retained in the attribute. (1 - highest)
Fax
Student Fax - Fax (data type INTEGER)

3. Student Account

Sections where some student account information is stored

UserName
Keeps user account login (data type STRING)
Created
Date when account was created (data type DATE)
Updated
Date of last change (Ddata type DATE)
LastLogged
The date the user was last logged into the system (data type DATE)
VerifiedUser
Flag if the registered user account is properly authenticated (data type BOOL)
BlockedUser
Flag if user account is blocked (data type BOOL)
ISICnumber
ISIC number of student (data type INTEGER)
LicensePlate
LicensePlate of student´s car for parking (data type STRING)

4. Student Information

The sections where the student's study information is stored

Degree
Indicates the level of study the student is studying (own data type DEGREE)
TypeofStudy
Indicates the type of study (own data type TYPY_STUDY)
Class
Year in which the student is located (own data type CLASS)
Faculty
Abbreviation of the faculty to which the student is studying (own data type FACULTY)
Language
Language in which student is studying (data type STRING)
Field
The field students are studying (data type STRING)
Grades
Student AVG grade (data type FLOAT)
